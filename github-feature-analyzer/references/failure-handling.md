# 失败处理

## 硬停止条件

遇到以下情况立即停止，不继续猜测：

- `repo_url` 无效。
- 仓库不是 `github.com`。
- 仓库是私有仓库或需要认证。
- `features` 为空。
- archive/API 和 `git clone` 都无法获取源码。
- 源码目录为空或无法索引。
- 开启历史检索时，Python venv 创建、依赖安装、向量模型或索引加载失败。

## 可恢复条件

下面情况可以降级继续，但必须在报告里说明：

- README 缺失或信息太弱：改用代码索引推断项目特征，并标记 `inference`。
- MCP 不可用：用脚本拉取源码，并说明跳过 MCP 探测。
- 多 agent 不可用：询问用户是否改用单 agent。
- UV 不可用：改用 `python3 scripts/reference_retrieval_local.py` 的纯 Python 自举路径。
- 某个子 agent 输出不合法：要求重跑一次；仍失败则记录缺口并继续合并其他结果。
- 某个功能证据不足：标记为低置信度，不要写成确定结论。

## 报告失败信息

失败时返回：

- 失败阶段
- 直接错误信息
- 已完成的步骤
- 可恢复建议
- 已生成的产物路径，若有

## 不允许的行为

- 不要把无法获取的私有仓库伪装成公开仓库。
- 不要编造文件路径、行号或实现细节。
- 不要在没有证据时输出高置信度结论。
- 不要自动清理下载的仓库，除非用户明确要求。
